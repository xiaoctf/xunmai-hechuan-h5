# UI参考与设计取舍

更新时间：2026-06-28

## 一、参考对象

本项目参考的是“非遗/文博/文旅数字展示”类网站的信息组织与视觉气质，不复制其图片、文字或具体版式。

| 参考方向 | 参考网站/资料 | 可借鉴点 |
| --- | --- | --- |
| 国家级非遗资料库 | 中国非物质文化遗产网·中国非物质文化遗产数字博物馆 | 资料库式栏目、项目梳理、专业可信的内容口径 |
| 沉浸式文化遗产展示 | 数字敦煌 | 首屏氛围、文化场景感、深色沉浸式视觉 |

| 非遗与旅游融合政策 | 文化和旅游部关于推动非遗与旅游深度融合发展的通知 | 非遗项目梳理、融入旅游空间、特色线路、体验基地 |
| 数字赋能传播策略 | 黑龙江省文旅厅“数字赋能下的非遗+文旅沉浸式体验设计与传播策略” | 真实性、互动性、传播效果评估、短视频/多渠道传播 |

## 二、本项目采用的设计取舍

### 1. 不做学校活动宣传页

实体物料和页面首屏优先呈现“合川非遗数字导览”的地方文化属性。团队与“三下乡”成果属性后置到底部落款，避免二维码扫开后第一眼像活动报名页或学校宣传页。

### 2. 移动端做 H5 故事流

移动端主要面向扫码访问，采用 9 屏纵向叙事：首页、项目导语、非遗资源库、匠人故事、技艺流程、文旅路线、助农产品、文创展示、团队落款。

### 3. 桌面端做展览导览站

桌面端主要面向老师电脑查看、投屏汇报和后续展示，采用横向信息密度更高的展览站结构：顶部导航、沉浸式首屏、栏目化内容、路线板块、成果落款。

### 4. 自动识别设备

入口页 `index.html` 自动识别访问设备：

- 手机或触摸设备进入 `mobile.html`。
- 桌面设备进入 `desktop.html`。
- 保留锚点，例如 `index.html#route` 会自动进入对应端的路线板块。

### 5. 非遗项目保持“待调研确认”

参考官方非遗网站的专业口径，当前不提前写死具体非遗名录。实地走访和资料核验后，再替换为真实项目、图片和访谈内容。

## 三、后续可升级方向

1. 用真实照片替换当前抽象场景视觉。
2. 给每个非遗项目建立单独详情卡。
3. 增加传承人音频或短视频入口。
4. 将路线图接入真实地图截图或手绘地图。
5. 将问卷二维码替换为正式问卷链接，并把统计结果做成看板。
